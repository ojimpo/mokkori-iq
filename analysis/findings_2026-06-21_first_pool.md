# 初プール採取の解析所見 — 2026-06-21

対象: `data/swim/swim_20260621_171349.csv`（股間装着・実機初の実データ）
- 46.3分 / 144,360サンプル / 52.1Hz / |acc|平均 ≒ 1g（重力健全、ドロップなし）
- 採取者の意図: 100m × 5本 × 2セット ＝ 1000m を大休憩でハーフ分割

## 1. この泳者の泳ぎ方（検出の前提）
- **オープンターン**（クイックターン不可）。毎回プールの底に足をついて立ってから壁を蹴る。
- **100mごとの休憩は水から上がらず数秒だけ静止してトントン(3タップ)→すぐ蹴り出す**（プール内で立ったまま）。
- **ベンチに座る長休憩はハーフ間の1回だけ**（数分、前後に約10秒の立位/歩行で出入り）。
- 実際の泳ぎ方の詳細は `data/swim/swim_20260621_171349.notes.md`（セッション記録＝疑似GT）。
- ペース体感 ≒ 30s/25m（解析と一致）。

## 2. 突破口: 重力z成分（gz）による壁検出
立位/着座で股間デバイスの体軸が垂直化し、**重力のz成分が「泳ぎ -7.5 ↔ 立位 +9.5 m/s²」と大きく振れる**。
これが最も明確な壁シグネチャだった。比較した他手法は不適:
- グライドギャップ（プッシュオフ後の静止）: 25m周期が自己相関で立たず、ストロークリズムに埋もれる。
- フリップ回転: 全セッションで大回転イベントは11個のみ（オープンターンなので宙返り回転が無い）。
- jerkインパルス（タップ検出用）: 泳ぎ/壁タッチに常時埋もれ、復元不能。

検出器 `analysis/turn_stand.py`:
- gz>2 の立位エピソードを分類。**ピークgz>5** で「半立ち/水中ボビング」を除外、**不応期12s**で過剰カウント抑制。
- `turn`(1.2–7s)=途中ターン壁 / `rest`(7–90s)=**100m境界（水中の数秒静止+トントン）** / `big-rest`(>90s)=ハーフ分割。
- lengths = walls + 1（25mプール: 100m = 4レングス = 3ターン）。
- **境界は2信号で立つ（実証）**: 立位エピソード継続時間が **途中ターン≒3秒(タップ無し)** と **100m境界≒7–17秒(トントン有り)** に明確分離。
  本人体感「数秒」でも、減速+静止+トントン+蹴り出しを含むので境界は実際 7–17s。トントンは無駄でなく有効（立位ゲートで jerk 三連が長い静止と一致）。

## 4. 3タップマーカーは gz 領域で復元できた
jerk領域では不能だったが、**立位で打った 0.9s × 3連**は gz-立位の三連ブリップとして分離できた（901s / 938s）。
→ 採取手順の改訂: タップは「壁で足をついて立位で」打つと gz で拾える見込み。

## 5. ストローク率（SPM）も計測可 — 当初想定外の収穫
股間は体ロールのリズムが乗る。泳ぎ区間でジャイロのロール軸のスペクトルピーク＝ストロークサイクル率。
- **両半とも安定して 30 cyc/min（≒60 arm strokes/min）**。3手法一致（窓FFT29 / 自己相関28-29 / トラッカー30）。
- 注: |acc|/|gyro| は整流で2倍/4倍harmonicsが出るので符号付き単軸で取る。
- 8s窓は分解能0.125Hz=7.5cyc/min刻みに量子化（精度が要るなら自己相関/放物線補間/長窓）。
- ツール: `analysis/stroke_rate.py`。

## 6. 本質的な制約: 外部グラウンドトゥルースが無い
プールにスマホ持込不可＋本人も覚えていられない（＝だからこのデバイスを作っている）。3タップが唯一のGT機構だったが
水中タップで失敗。よって検出は **内的整合性（ターン等間隔）＋妥当ペース** で検証するしかない。前半が意図500mと一致したのは強い傍証。

## 7. 次段への含意
- 再チューニングは「手首/30Hz の |acc|/activity FSM」ではなく **gz-オリエンテーション壁検出を主軸**に。
- gz が効くのは装着向き（USB-C上/Seeed面外側）前提。向き未知なら「泳ぎ⇄静止で最も振れる軸」を自動選択する一般化が必要。
- 採取手順: タップは立位で。後半のような連続泳ぎでもレップ境界を残すなら、各本後に一度しっかり立つ運用が望ましい。

## 8. 一発レポート化（デバイス完成形） & 保守バイアス
`analysis/swim_report.py <csv>` で取込CSV→距離/ペース/レップ内訳/SPM/Strava用サマリを一発出力。
**距離・ペースは保守側に倒す方針**（過大NG・過小OK / 速めNG・遅めOK）を実装:

    本数 = min(壁カウント, round(泳ぎ時間 / 標準レングス時間))     # レップごと

- 偽分割（壁を二重カウント）で増える分を ペース項 が頭打ち。
- 壁取りこぼしは 壁カウント が低く出る→ min がその低値を採用＝過小（許容方向）。
- 綺麗なレップは両項一致で正確に保つ（検証済み100mを floor で削らない）。
- 標準レングス時間＝「ターン間の泳ぎ実時間(立位除外)」の中央値＝28.5s/25m（分子の泳ぎ時間と単位を揃えるのが肝。
  start-start間隔だと立位3sが混入して35sになり綺麗な100mを誤って削る、という初期バグを修正済み）。

**最終出力（このセッション, 保守版）**:

|---|---|---|
| 前半 | 475m | 2:01 |
| 後半 | 675m | 1:53 |
| **合計** | **1,150m** | **約1:57**（moving 22:20 / 総経過 46:16）|

レップ内訳: 半1=100/100/100/175m、半2=100/325(連続)/50/200m。前半の綺麗な3本は壁=ペース=4で100m保持、
後半 rep5 は偽分割を min で 9→8本に補正。ストローク率 約30 cyc/min。タップ境界 901/938s を gz で復元。
