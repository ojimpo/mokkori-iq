# Step 2: データ探索の発見まとめ

Brunner et al. (ISWC 2019) データセット（手首装着スマートウォッチ, 30Hzリサンプル）の
自由形ターンを中心に信号特徴を分析した結果。

## データセット構造

- 40被験者 / 206セッション / 合計 約18時間
- 泳法別セッション数: Freestyle 92, Butterfly 58, Backstroke 33, Breaststroke 23
- 自由形にフォーカス: 92セッション・30被験者・*310ターン*
  - ターン/セッション: 中央値3（範囲0〜16）
  - 21セッションはターン0個（精度＝誤検出評価に使える）
- カラム `ACC_012` は加速度3軸のL2ノルムが事前計算済み（自前計算と一致、誤差1e-15）
- `label`: 0=null(休憩), 1=freestyle, 2=breaststroke, 3=backstroke, 4=butterfly, 5=turn
- 重要: label=5（ターン）は瞬間イベントではなく*幅の広い遷移区間*
  - ターン区間長（秒）: p5=2.5, 中央値=4.1, p95=6.5, 最大13.2
  - → 1ターン＝この窓内に検出を1個出せば正解、という評価設計にする

## 加速度の素性

- 加速度は重力込み（Android TYPE_ACCELEROMETER）。静止時ノルム ≈ 9.8 (1g)
- クラス別 acc_norm 中央値: null=9.9, freestyle=12.8(p95=27.6), turn=10.2

## ターンの信号 signature（核心）

「ストローク減速 → 壁タッチで静止 → 壁キックで加速」という仮説を activity envelope
（acc_normの移動標準偏差, 窓0.5秒）で検証した。

| 特徴 | 値（自由形310ターン） | 解釈 |
|---|---|---|
| 泳ぎ中の envelope（セッション中央値） | 4.84 | 定常ストロークの活動量 |
| ターン窓内の谷 = env最小 | 中央値 0.47 (p25=0.35, p75=0.65) | 泳ぎの約1/10に急落＝静止 |
| キックスパイク = acc_norm最大 | 中央値 25.7 (p25=21.8) | 壁押し出しの加速ピーク |
| 静止区間長（env<0.4×泳ぎ） | 中央値 1.27秒 (p25=0.80, p75=1.69) | 壁タッチ〜キックの低活動 |
| 静止区間を持つターンの割合 | 98% | 極めて頑健な特徴 |

## 設計への示唆

1. 主特徴は「低活動の谷」。envが「最近の泳ぎ活動」の30〜40%を下回り、
   かつ短時間（〜1.3秒）で泳ぎに戻る区間がターン。位置非依存で全泳法に存在
   → Phase1で股間装着に移っても効くはず。
2. 休憩（null）との区別: 休憩も低活動だが「長い」。谷の継続が一定上限（〜4秒）を
   超えたら休憩としてターン計上しない（最大継続時間で弁別）。
3. キックスパイク（acc_norm > ~20）は補助確認に使える。
4. ジャイロは補助: タッチターンは前転しないので大回転は無い。主軸は加速度の谷。
5. 適応閾値: 閾値は「最近の泳ぎ活動」のピークフォロワーに対する比率で持つ。
   装着位置・被験者で絶対スケールが変わっても追従できる（Phase1移行のため）。
6. causal設計: 谷に入る→泳ぎ活動が戻った瞬間に「直前に谷があった」と確定して
   過去タイムスタンプ（谷の最小点）を発報。未来参照なし・有限メモリ（リングバッファ数秒）。

→ 有限状態機械（SWIM→DIP→発報）で実装する。閾値は config に外出し。
